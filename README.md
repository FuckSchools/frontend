# FuckSchools Frontend

## Purpose
The Studio Orchestrator. Renders normalized backend signals and reflects traversal changes.

## Studio Orchestrator
Coordinates the live learning surface: selected node, viewport, blocker panels, and interaction state.

## Multi-modal rendering layer
- Graph mode: Tree and traversal.
- Spatial mode: 3D/motion-first presentation.
- Inspection mode: Blockers and completion criteria.
- Interaction mode: Accept, reject, branch, revisit.
